## Feature Modules

- `auth`: signup/login/refresh/verify/logout + Google OAuth entry points
- `users`: protected user retrieval/list endpoints
- `health`: API, DB, readiness checks
- `sessions`: session lifecycle abstraction over session repository
- `rbac`: role-to-ability policy service

## Layering Rules

- Controller: transport and route decorators
- Service: business logic and orchestration
- Repository: Drizzle data access

Keep this split consistent for new features.

## Cross-Cutting Components

- Guards:
  - `JwtAuthGuard`
  - `RolesGuard`
  - `AbilitiesGuard`
- Decorators:
  - `@Public()`
  - `@Roles(...)`
  - `@RequireAbilities(...)`
- Global filters:
  - `HttpExceptionFilter`
  - `AllExceptionsFilter`

## Database Area

- All table definitions and relation types live in `src/database/drizzle/schema.ts`.
- Migrations are generated by `drizzle-kit`.
- Database DI provider token: `DATABASE_PROVIDER`.

## API Docs Area

- Swagger setup: `src/swagger.ts`
- Swagger route: `/api-docs`
- Repo-facing docs: `docs/`
- Agent-facing docs/process notes: `agent-docs/`

## Path Aliases

Configured in `tsconfig.json` and used across source code:

- `@/*`, `@modules/*`, `@common/*`, `@database/*`, `@types/*`
- plus scoped aliases for decorators/exceptions/guards/constants and others

Prefer aliases over deep relative imports.
